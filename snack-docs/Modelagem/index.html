Glossario

  • Sistema - site com o qual os usuarios interagem para efetuar a compra/venda
  • Comprador - Algum usuario que esteja efetuando uma compra no sistema
  • Vendedor - Algum usuario que esteja efetuando uma venda no sistema
  • Pedido - Lista de produtos de um determinado vendedor que um comprador demonstra insteresse em comprar

Requisitos Não-funcionais

# Descrição Métrica
RNF001 Segurança As informações enviadas pelo site devem ser criptografadas
RNF002 Usabilidade O site deve se adapatar ao tamanho dos dispositivos que o acessam, funcionando em telas mobile com no minimo 640x360, e telas desktop com no minimo 1024x768
RNF003 Disponibilidade O site deve estar disponivel 24 horas por dia, 7 dias por semana
RNF004 Compatibilidade O site deve ser compativel com Chrome v68 ou superior, Safari v12 ou superior, Firefox v68 ou superior, IE v11 ou superior

Regras de negócio

# Descrição
RDN001 O site deve mostrar vendedores proximos ao comprador
RDN002 O usuario pode acessar a lista de produtos sem estar logado
RDN003 O usuario deve estar logado para efetuar uma compra
RDN004 O usuario deve cadastrar uma senha de no minimo 8 digitos alfanumericos
RDN005 O usuario deve cadastrar um email valido
RDN006 Os usuarios devem permitir o acesso do sistema a sua localização

Diagramas de caso de uso

Descrição dos atores

Usuario

Pessoa que esteja cadastrada no sistema

Comprador

Usuario com intenção de comprar um ou varios produtos no sistema

Vendedor

Usuario com intenção de vender um ou varios produtos no sistema

Descrição dos casos de uso

UC01 - Cadastrar Usuário

Descrição:

- O usuário deve se cadastrar para usar o sistema.

Ator:

- Vendedor

Pré-condições:

- O usuário deve ter email válido.

- O usuário acessar a página de cadastro de novo usuário.

Pós-condições:

- O usuário estará logado no sistema.

Diagrama de atividades:


UC02 - Cadastrar Produto

Descrição:

- O vendedor deve cadastrar produto para a venda.

Ator:

- Vendedor

Pré-condições:

- O vendedor deve estar logado no sistema.

- O vendedor acessar a página de cadastro de produtos.

Pós-condições:

- O vendedor terá produtos disponíveis para a venda.

Diagrama de atividades:


UC03 - Listar Produtos

Descrição:

- O comprador deve acessar uma lista dos produtos disponíveis para a compra.

Ator:

- Comprador

Pré-condições:

- O comprador estar logado no sistema.

- O comprador acessar a página de visualização de produtos.

Pós-condições:

- O comprador terá acessado os produtos disponíveis para selecionar.

Diagrama de atividades:


UC04 - Selecionar Produto

Descrição:

- O comprador deve acessar uma lista dos produtos disponíveis para a compra e selecionar um produto.

Ator:

- Comprador

Pré-condições:

- O comprador estar logado no sistema.

- O comprador acessar a página de visualização de produtos.

Pós-condições:

- O comprador terá selecionado um produto para a posterior compra.

Diagrama de atividades:


UC06 - Finalizar Pedido

Descrição:

- O comprador deve finalizar o pedido depois de selecionar os produtos desejados.

Ator:

- Comprador/Vendedor

Pré-condições:

- O comprador ter escolhido os produtos que deseja comprar.

Pós-condições:

- Um pedido de venda é enviado ao vendedor.

Diagrama de atividades:


UC07 - Fazer Pagamento

Descrição:

- O comprador deve efetuar o pagamento, no dinheiro, no ato da venda, ou por cartao, pelo site, ao fazer o pedido.

Ator:

- Comprador/Vendedor

Pré-condições:

- O comprador ter escolhido os produtos que quer comprar.

- O comprador confirmar a compra.

- O vendedor confirmar a entrega.

Pós-condições:

- O vendedor ter recebido o valor do pedido.

Diagrama de atividades:


UC11 - Login

Descrição:

- O usuário deve logar no sistema informando seu e-mail e senha cadastrados.

Ator:

- Usuário

Pré-condições:

- O Usuário estar cadastrado no sistema.

- O Usuário estar na tela de Login.

Pós-condições:

- O usuário deve ter acesso às funcionalidades do sistema.
